# Google OAuth 2.0 Setup Guide for NewsFlow

This guide explains how to configure and set up Google OAuth 2.0 Single Sign-On (SSO) for the NewsFlow application, integrating it with a centralized authentication server.

---

## Part 1: Getting Google Cloud Credentials

Before writing any code, you must register your application with Google to get a Client ID.

1. **Go to the Google Cloud Console**
   Navigate to [console.cloud.google.com](https://console.cloud.google.com/) and log in.

2. **Create or Select a Project**
   Use the project dropdown at the top to select an existing project or create a new one.

3. **Configure the OAuth Consent Screen**
   - Go to **APIs & Services > OAuth consent screen**.
   - Select **External** (or Internal if using a Google Workspace).
   - Fill out the required fields: App Name (e.g., "NewsFlow"), User Support Email, and Developer Contact Email.
   - Click **Save and Continue** through the remaining steps.

4. **Create the OAuth Client ID**
   - Go to **APIs & Services > Credentials**.
   - Click **+ Create Credentials > OAuth client ID**.
   - Set the Application type to **Web application**.
   - Set the Name to something recognizable (e.g., "NewsFlow Frontend").
   - Under **Authorized JavaScript origins**, add the URLs where your frontend runs:
     - `http://localhost:5173`
     - `http://localhost`
   - Click **Create**.
   - **Copy the generated Client ID.** You will need this for both the frontend and the central auth server.

---

## Part 2: Environment Configuration

Once you have your Client ID, you must configure the environment variables across the different parts of the application.

### 1. Frontend Configuration (`Frontend/.env.local`)
Create a `.env.local` file in the `Frontend` directory:
```env
# URL of your central authentication server
VITE_AUTH_URL=http://localhost:3115

# Your Google OAuth Client ID
VITE_GOOGLE_CLIENT_ID=your-google-client-id-here.apps.googleusercontent.com

# URL of your NewsLetter Backend
VITE_API_URL=http://localhost:5000/api
```

### 2. Backend Configuration (`Backend/.env`)
Update the `.env` file in your `Backend` directory:
```env
# Central Auth Server for token verification
AUTH_URL=http://localhost:3115

# Allowed Frontend Origin (for CORS)
FRONTEND_URL=http://localhost:5173
```

### 3. Central Auth Server Configuration (`auth.vjstartup.com` or `localhost:3115`)
**Crucial Step:** The central authentication server *must* also be updated to accept tokens generated by your new Client ID. Update its environment variables to include the new `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET`.

---

## Part 3: Running the Servers

For the authentication flow to work correctly locally, you must run all three servers simultaneously:

1. **Central Auth Server:** Running on port `3115` (or whatever `AUTH_URL` is set to).
2. **NewsLetter Backend:** Open a terminal in the `Backend` folder and run `npm run dev` (runs on port `5000`).
3. **NewsLetter Frontend:** Open a terminal in the `Frontend` folder and run `npm run dev` (runs on port `5173`).

---

## Troubleshooting

- **Error 400: origin_mismatch:** You forgot to add `http://localhost:5173` to the "Authorized JavaScript origins" in the Google Cloud Console.
- **Invalid Audience Error:** The token was accepted by Google, but rejected by your Central Auth Server. This means the Central Auth Server hasn't been updated with your new `VITE_GOOGLE_CLIENT_ID`.
- **CORS Errors:** Ensure your backend `app.js` is correctly configured to allow the `FRONTEND_URL` origin and has `credentials: true` enabled.
